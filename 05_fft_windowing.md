# 🌊 Module 5. FFT와 한(Hann) 윈도우: 스펙트럼 누설의 비밀과 해법

본 장에서는 시간 도메인의 이산 데이터를 주파수 도메인 상의 주파수 성분으로 분해하는 **고속 푸리에 변환(FFT, Fast Fourier Transform)** 과정에서 발생하는 불가피한 물리적 왜곡인 **스펙트럼 누설(Spectral Leakage)**의 근본 원인을 규명합니다. 또한 이를 보정하기 위해 사용되는 **한 윈도우(Hann Window)**의 수학적 원리와 임베디드 신호처리 시스템에서의 학술적·공학적 트레이드오프(Trade-off)를 상세히 설명합니다.

---

## 1. 푸리에 분석의 직관적 해석

시간 축 상의 복합적인 아날로그 물리 신호(예: 마이크로폰을 통해 입력되는 목소리)를 주파수 스펙트럼 상으로 변환하는 푸리에 변환(Fourier Transform)은 혼합 과일 스무디를 화학적으로 분해하여 개별 과일 원료의 성분 비율표를 도출해내는 과정과 유사합니다.

```
 [ 시간 도메인: 복합 신호 ]                 [ 푸리에 변환 (FFT) ]            [ 주파수 도메인: 스펙트럼 ]
   / \   /\       /\
  /   \_/  \/\_  /  \       =============>       |       |
 /             \/    \                           |  딸기 |  바나나  |  우유
+----------------------+                         +-------+--------+-------+
    (과일들이 섞인 스무디)                              (성분별 레시피 출력)
```

디지털 연산 장치는 이 스펙트럼 성분 추출을 위해 **이산 푸리에 변환(DFT, Discrete Fourier Transform)** 또는 이를 고속화한 알고리즘인 **FFT(Fast Fourier Transform)**를 사용합니다. 이때, 복소수 표기 체계를 기반으로 데이터를 처리합니다.

### 🧩 복소 주파수 성분의 물리적 의미
* **실수부 (Real Part, $Re$):** 입력 신호가 기준 신호인 **코사인(Cosine)** 파형과 지니는 상호 상관관계(유사도)를 정량화합니다.
* **허수부 (Imaginary Part, $Im$):** 입력 신호가 기준 신호인 **사인(Sine)** 파형과 지니는 상호 상관관계를 정량화합니다.
* **스펙트럼 크기 (Magnitude, $|X| = \sqrt{Re^2 + Im^2}$):** 
  입력 신호의 기입 타이밍(위상각, Phase)이 지연됨에 따라 실수부와 허수부의 개별 상관 계수는 변화합니다. 그러나 두 값을 각각 제곱하여 합산한 뒤 제곱근을 계산하면, 시간적 지연 오프셋에 영향받지 않는 **해당 주파수 성분의 고유한 순수 에너지(진폭)**를 획득할 수 있습니다.

## 2. 시간 도메인 단절과 스펙트럼 누설 (Spectral Leakage)

디지털 시스템은 물리적 버퍼 메모리 및 연산 속도의 한계로 인해 무한한 시간의 신호를 연속 처리할 수 없습니다. 따라서 실무 임베디드 오디오 코어는 항상 **특정 표본 수($N$개)로 구획된 시간 프레임**을 분할하여 FFT를 연산하게 됩니다. 이 유한한 시간 영역의 절단 과정에서 왜곡이 파생됩니다.

### ⚠️ 정수 배 주기성과 불연속적 단절의 비교
DFT 연산은 잘라낸 $N$개의 시간 샘플이 **시간 도메인 상에서 무한히 반복하여 교차하는 주기 신호**라고 가정하여 수학적 수렴식을 계산합니다.

1. **정수 배 주기를 형성하는 프레임:**
   잘라낸 분석 버퍼의 맨 시작점(Start Boundary)과 끝점(End Boundary)의 순시 진폭이 매끄럽게 연결됩니다. 이 경우 주파수 결과는 에너지 퍼짐 현상 없이 날카로운 하나의 델타 기둥으로 정확하게 표상됩니다.
2. **비정수 배 주기를 형성하는 프레임:**
   시작 경계와 끝 경계의 순시 진폭 높이가 일치하지 않습니다. 이 프레임을 무한 주기로 이어 붙일 때, 경계 교차점마다 툭툭 끊어지는 **급격한 수학적 단절(Discontinuity/불연속성)**이 관측됩니다.

```
[ 정수 배 주기: 매끄러운 연결 ]               [ 비정수 배 주기: 경계면의 툭 끊김 (불연속성) ]
    / \     / \     / \                          / \     / \     / |
   /   \   /   \   /   \                        /   \   /   \   /  |
--+-----+-----+-----+-----+--                 --+-----+-----+-----+ +-----+
  |<--- 관측 구간 N --->|                       |<--- 관측 구간 N --->| | (툭 끊김!)
```

### 🌋 스펙트럼 누설의 기하학적 유도
시간 도메인 상에서의 급격한 진폭 단절(Discontinuity)은 수학적으로 매우 높은 주파수의 고조파 성분을 무한히 유도합니다.
이는 신호에 불연속 경계를 지닌 **직사각형 창(Rectangular Window)**을 임의로 곱한 것과 같으며, 직사각형 창의 날카로운 에지 부분이 주파수 도메인 상에서 디렉 델타 함수 모양의 단일 주파수 주변으로 질질 흘러내려 잡음 언덕을 형성하게 됩니다. 이 현상을 **스펙트럼 누설(Spectral Leakage)**이라 칭합니다.

```
          [ 사각형 창(Rectangular Window)을 거친 FFT 결과 ]

                 진폭 (dB)
                    ^
                    |          | <-- 원래 존재해야 하는 단일 주파수
                    |         / \
                    |        /   \
                    |       /     \
                    |      /       \  <-- 에너지가 양옆으로 질질 흘러내림
                    |     /         \_ _ _ _
                    +----+----+----+----+----> 주파수 (Frequency)
```

## 3. 창 함수 조절을 통한 스펙트럼 누설 제어: 한 윈도우 (Hann Window)

시간 축 절단에 의한 누설 왜곡을 제어하기 위해, **"유한 구간의 양쪽 경계면 진폭 수준을 수학적으로 부드럽게 감쇄시켜 0으로 수렴하게 만들자!"**는 주파수 윈도잉 기법을 사용합니다. 이를 위한 대표적인 수학적 창 함수가 **한 윈도우(Hann Window)**입니다.

> [!NOTE]
> **창 함수의 역사적 정의**
> 이 창 함수는 오스트리아의 기상학자 Julius von Hann의 스무딩 기법 연구에서 고안되었습니다. 일부 문헌에서 '해닝 윈도우(Hanning Window)'로 혼용하여 번역되기도 하나, 공식 표준 학술 명칭은 **한 윈도우(Hann Window)**입니다. 이는 해밍(Hamming) 윈도우와 명확히 구분되어야 합니다.

### 📐 한 윈도우의 수학적 거동식: 대칭형(Symmetric) vs 주기형(Periodic)

한 윈도우는 좌우 대칭인 완만한 종 모양의 형태를 보이며, 수학적으로는 코사인 신호를 공중에 띄운 **Raised Cosine** 관계식으로 기술됩니다. 임베디드 오디오 처리 시스템에서 한 윈도우는 목적에 따라 **대칭형(Symmetric)**과 **주기형(Periodic)**으로 세분화되어 적용됩니다.

#### 1) 대칭형 (Symmetric) 한 윈도우
$$w[n] = 0.5 \cdot \left( 1 - \cos\left( \frac{2\pi n}{N-1} \right) \right) \quad (0 \le n \le N-1)$$
* **구조적 특징:** 분모가 $N-1$로 설계되어 $n=0$과 $n=N-1$에서 가중치가 정확히 **$0.0$**으로 떨어지며, 수학적으로 완벽한 좌우 대칭 구조를 형성합니다.
* **주요 용도:** 단일 프레임 주파수 분석(Spectrum Analysis)이나 FIR 필터 계수 설계 시 주로 채택됩니다.

#### 2) 주기형 (Periodic) 한 윈도우
$$w[n] = 0.5 \cdot \left( 1 - \cos\left( \frac{2\pi n}{N} \right) \right) \quad (0 \le n \le N-1)$$
* **구조적 특징:** 분모를 $N$으로 설정하여 시작 원소 $w[0]$은 $0.0$이 되지만 끝 원소 $w[N-1]$은 $0.0$이 아닌 약간의 양수 값을 갖게 만듭니다. 대신, 다음 프레임의 첫 원소 $w[N] \equiv w[0] = 0.0$과 부드럽게 이어지는 주기적 대칭성을 보장합니다.
* **주요 용도:** 단시간 푸리에 변환(STFT) 분석 및 오디오 역변환(iFFT) 단계에서 필수적으로 활용됩니다. 홉 크기(Hop Size) 단위로 프레임을 중첩하여 더하는 **중첩 가산(OLA, Overlap-Add) 과정**에서 시간축상의 총 에너지 합이 항상 일정(Constant-OLA)하게 유지되는 **완벽 복원(Perfect Reconstruction)** 요건을 충족하기 위해서는 주기형(Periodic) 윈도우를 사용해야 합니다.

```
      Symmetric (분모: N-1)                   Periodic (분모: N)
      1.0 +     *****                        1.0 +     *****
          |   **     **                          |   **     **
          |  *         *                         |  *         *
      0.0 +**-----------**+                  0.0 +**-----------* +   (w[N-1] > 0)
          0            N-1                       0            N-1
       (양 끝이 완벽히 0)                           (다음 프레임과 매끄럽게 연결)
```

### 🔄 시간축 상의 합성 연산
고속 푸리에 변환(FFT)을 수행하기 전, 임베디드 프로세서는 시간 버퍼에 적재된 원래 신호 표본 $x[n]$에 한 창 함수의 각 가중치 계수 $w[n]$을 원소 곱(Element-wise Multiplication) 연산합니다.

$$x_{windowed}[n] = x[n] \times w[n]$$

이 윈도잉 연산을 통해 512샘플의 중앙부 진폭 가중치는 완전하게 확보하되, **양끝의 가중치 $w[0]$ 및 $w[N-1]$은 0으로 완벽히 수렴**하게 됩니다. 이 덕분에 구획된 신호가 시간축 상에서 무한 반복되더라도 어떠한 수학적 불연속 단절점도 형성되지 않습니다.

```
  [ 원래 신호 (비정수 배 주기) ]            [ 한 윈도우 (Hann) ]            [ 윈도잉 적용 후 신호 ]
       / \   /\   /\                         .---.                       .---.
      /   \_/  \_/  \                      /       \                   /   \_/   \
  ---+---------------+---        X        +         +       ======> --+-----------+--
     |<--  N 샘플  -->|                    \       /                  |<-- N샘플 -->|
    (양 끝의 높이가 다름)                   (양 끝이 0)                 (양 끝이 0으로 부드러움)
```

## 4. 주엽(Mainlobe)과 부엽(Sidelobe)의 공학적 트레이드오프

윈도잉을 통하여 주파수 누설을 극적으로 억제하는 것은 공짜로 얻어지지 않으며, 스펙트럼 분석상의 **주엽(Mainlobe)** 대역폭과 **부엽(Sidelobe)** 레벨 사이에 뚜렷한 상보 관계(Trade-off)가 형성됩니다.

```
                               주엽 (Mainlobe)
                                  |  |
                                  \  /
                           ________\/________
                          /                  \   <-- 부엽 (Sidelobe)
                         /                    \
```

### 1) 메인로브 (Mainlobe, 주엽)의 대역폭
* **물리적 현상:** 시간 도메인의 양끝 정보를 0 근방으로 깎아냄에 따라 주파수 상에서는 원래 임의의 주파수 성분을 묘사하는 중심 기둥의 폭이 직사각형 창에 비해 **2배 두껍게 팽창합니다** (Peak-to-Null 반폭 기준, Rectangular: $\frac{2\pi}{N} \to$ Hann: $\frac{4\pi}{N}$).
* **시스템 영향:** 주파수 해상도(Frequency Resolution)가 하락합니다. 즉, 주파수 간격이 매우 조밀한 두 인접 오디오 성분이 하나의 넓은 주엽 기둥으로 합쳐져 상호 분별이 불가해질 수 있습니다.

> [!NOTE]
> **주엽 폭(Mainlobe Width) 산정 기준 안내**
> 아래 비교 표에 표기된 주엽 폭 수치는 중심 주파수에서부터 첫 번째 주파수 영점(Null Point)까지의 거리인 **Peak-to-Null (Half-width, 반폭)** 기준입니다.
> 양측 첫 번째 Null 사이의 전체 대역폭을 뜻하는 **Null-to-Null (Full-width, 전폭)** 기준으로 표현할 경우, 사각형 창은 $\frac{4\pi}{N}$, 한 창 및 해밍 창은 $\frac{8\pi}{N}$가 됩니다.

### 2) 사이드로브 (Sidelobe, 부엽)의 감쇄율
* **물리적 현상:** 시간축 상의 연속성 확보로 인하여 중심 주엽 기둥 바깥으로 누설되는 에너지 찌꺼기가 극대 수준으로 억제됩니다. 부엽의 최고 진폭 레벨은 사각형 창($-13\text{ dB}$) 대비 **$-32\text{ dB}$ 이하로 감쇄**됩니다.
* **시스템 영향:** 큰 주파수 성분 옆에 숨어있는 **극미세 신호 성분의 에너지를 손실 없이 탐지해 낼 수 있습니다.** 사각형 창의 경우 미세 신호가 인접 고전력 주파수의 누설 부엽 찌꺼기에 덮여 유실됩니다.

| 창 함수 형식 (Window Type) | 주엽 폭 (Mainlobe Width, Peak-to-Null 반폭 기준) | 최대 부엽 수준 (Max Sidelobe Level) | 주요 공학적 용도 |
| :--- | :--- | :--- | :--- |
| **사각형 창 (Rectangular)** | $\frac{2\pi}{N}$ (매우 예리함) | $-13\text{ dB}$ (매우 높음) | 표본 길이가 주기 정수 배와 정합되는 기저 신호 분석 |
| **한 창 (Hann)** | $\frac{4\pi}{N}$ (2배 확산) | $-32\text{ dB}$ (우수한 억제력) | 범용 음성/오디오 대역 신호분석 및 통계 노이즈 검출 |
| **해밍 창 (Hamming)** | $\frac{4\pi}{N}$ (2배 확산) | $-43\text{ dB}$ (첫 부엽이 가장 낮음) | 선형 예측 기동 음성 인식 및 주파수 필터 계수 설계 |

## 5. 임베디드 윈도잉 및 스펙트럼 분석의 Python 참조 구현

다음 파이썬 소스코드는 한 윈도우 가중치 룩업 테이블(LUT)을 계산하여 입력 신호 벡터에 매핑한 뒤, 복소 푸리에 변환 스펙트럼으로부터 전력 진폭 강도를 도출하는 단계를 검증합니다.

```python
import numpy as np

def apply_hann_window_and_fft(signal_buffer):
    """
    유입된 시간축 오디오 버퍼에 한 윈도우를 적용하고
    FFT를 처리하여 순수 진폭 크기 성분을 산출합니다.
    """
    N = len(signal_buffer)
    
    # 1. 한 윈도우 함수 LUT 계산 (임베디드 타겟 플래시 메모리에 사전 정의 보관 가능)
    hann_window = 0.5 * (1.0 - np.cos(2 * np.pi * np.arange(N) / (N - 1)))
    
    # 2. 시간 도메인 입력 벡터에 창 가중치 적용 (원소 간 곱셉)
    windowed_signal = signal_buffer * hann_window
    
    # 3. 고속 푸리에 변환 수행
    fft_result = np.fft.fft(windowed_signal)
    
    # 4. 복소 스펙트럼 벡터로부터 순수 실수 진폭(Magnitude) 산출: sqrt(Re^2 + Im^2)
    magnitude = np.abs(fft_result)
    
    return magnitude
```

## 6. 실시간 임베디드 의료기기(인공와우) 설계 시의 윈도잉 제약과 대책

샘플링 주파수 $16\text{ kHz}$, FFT 크기 512포인트, 홉 사이즈 16샘플의 실시간 연속 필터링을 지닌 인공와우 설계 시 일반적인 대칭형 윈도잉은 두 가지 치명적인 물리적 상쇄 왜곡을 초래합니다.

### A. 시스템 기동 초기 과도 상태(Transient Phase)의 에너지 소실 문제
* **물리 현상:** 시스템 전원을 켜고 초기 1ms 시점에는 버퍼의 앞 16칸에만 유효 신호가 차고 나머지 496칸은 제로패딩 상태를 가집니다. 이 조건 하에서 대칭형 Hann 창 함수를 곱해주면, 데이터가 존재하는 버퍼 좌측($n = 0 \sim 15$) 구간의 Hann 계수 $w[n]$이 거의 $0$ 수준(예: $w[0]=0$, $w[15] \approx 0.026$)을 보입니다. 결과적으로 유효한 외부 오디오 정보가 강제로 지워지는 기동 초기 에너지 증발 왜곡이 발생합니다.
* **해결 방안:** 
  * **직사각형 창 우회 (Bypass):** 유효 프레임이 충분히 유입되기 전(예: 누적 표본 수 $M < 128$)에는 임시로 Hann 창 함수 적용을 바이패스(가중치 $1.0$ 유지)하고 데이터 유효 점수에 따른 이득 스케일 보정 계수만을 연산하여 초기 지연을 1ms 범위로 제어합니다.
  * **가변 길이 동적 윈도잉 (Dynamic Windowing):** 유효 표본 크기 $M$의 변화에 정합하는 $M$-point Hann 창 함수를 동적으로 합성하여 윈도잉을 수행한 뒤 뒤편을 제로패딩합니다.

### B. 군지연 저감을 위한 비대칭 창 함수(Asymmetric Window) 적용
* **물리 현상:** 대칭형 Hann 창 함수를 적용하면 에너지의 중심축이 프레임의 중앙(256번째 샘플)에 자리 잡게 되어, 다음식과 같이 최소 $16\text{ ms}$의 시스템 군지연을 초래합니다.
  $$\text{Latency}_{\text{group}} = \frac{N}{2 \cdot f_s} = \frac{256}{16,000\text{ Hz}} = 16\text{ ms}$$
  이 수준의 대기 지연은 환자의 시·청각 결합 인지 한계를 초과합니다.
* **해결 방안:** 스펙트럼 에너지 분포 곡선의 왜도를 인위적으로 비틀어 피크 가중치가 가장 최근 입력 데이터(512포인트 버퍼의 가장 우측단)에 인접하여 생성되도록 최적 설계한 **비대칭형 창 함수(Asymmetric Window)**를 구현합니다. 이를 통하여 주파수 차단 성능을 그대로 보존하면서 실제 전달 군지연을 **$4 \sim 8\text{ ms}$** 범위로 감축합니다.

```
 [ 일반 대칭 윈도우: 16ms 지연 ]              [ 비대칭 윈도우: 4~8ms 지연 ]
            *                                              *
         *     *                                        *     *
       *         *                                    *         *
     *             *                                *             *
  ***---------------+***                         ***-----------------+*
  0                511                           0                  511
    (피크가 중앙에 위치)                          (피크가 최신 데이터(우측)에 편향)
```
